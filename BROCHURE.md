# sim-tooling

In one line: the tooling bench that keeps SIM's generated docs, contracts, and repo checks in sync.

## What it gives you

One place for the practical tools that keep the constellation legible. It builds
the generated contract files, checks repository shape, tracks public docs
freshness, projects the public SIM Index into managed Markdown vault namespaces,
and gives every code repo the same small set of maintenance commands. That means
a repo can prove what it publishes without inventing a private toolset or
copying scripts from its neighbors.

## Why you will be glad

- Public docs and contract files are generated the same way in every repo.
- Index evidence can be exported to portable, Obsidian, SeqLog, and Logseq
  Markdown vaults without creating a second public docs tree.
- Repository checks catch drift before it reaches the front page.
- Shared maintenance commands stay close to the crates they inspect.

## Where it fits

It is the workshop behind the public constellation: not the runtime, not a user
surface, but the set of repo-local tools that makes the runtime's published
shape inspectable and repeatable.
